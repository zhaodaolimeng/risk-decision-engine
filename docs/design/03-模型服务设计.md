# 模型服务设计

## 1. 概述

模型服务模块负责与外部模型服务对接，实现：
- 接收 `contractId`（合同ID）
- 获取决策流程中预加载的上下文数据
- 构造请求并调用外部模型服务
- 解析模型返回结果并映射到决策上下文中

---

## 3. 模型配置

### 3.1 配置结构

```yaml
# 模型配置示例
model:
  id: "M001"
  name: "违约预测模型"
  description: "预测用户违约概率"
  type: "HTTP"
  version: "1.0"
  status: "ACTIVE"

  # HTTP 配置
  config:
    url: "http://model-service:8000/api/v1/predict/default"
    method: "POST"
    timeout: "2s"

    # 请求头
    headers:
      Content-Type: "application/json"
      X-API-Key: "${MODEL_API_KEY}"  # 环境变量注入
      X-Request-ID: "${requestId}"

    # 输入映射（模板）
    # 支持的变量:
    #   - ${contractId}: 合同ID
    #   - ${requestId}: 请求ID
    #   - ${businessId}: 业务ID
    #   - ${input.*}: 请求输入数据
    #   - ${context.*}: 预加载的上下文数据
    inputMapping:
      contract_id: "${contractId}"
      applicant:
        age: "${context.applicant.age}"
        income: "${context.applicant.monthlyIncome}"
        education: "${context.applicant.education}"
      credit:
        queryCount3M: "${context.creditData.queryCount3M}"
        overdueCount2Y: "${context.creditData.overdueCount2Y}"
      extra:
        requestTime: "${__timestamp__}"  # 内置变量

    # 输出映射
    outputMapping:
      defaultProbability: "${data.default_probability}"
      riskScore: "${data.risk_score}"
      riskLevel: "${data.risk_level}"
      modelVersion: "${data.model_version}"

    # 重试配置
    retryConfig:
      maxAttempts: 3
      retryDelay: "100ms"
      retryableErrors:
        - "5xx"
        - "timeout"
        - "connection_error"

    # 降级配置
    fallback:
      enabled: true
      defaultOutput:
        defaultProbability: 0.5
        riskScore: 500
        riskLevel: "MEDIUM"
```

## 4. 接口契约

### 4.1 模型服务请求（标准）

```json
POST /api/v1/predict/{modelName}
Content-Type: application/json
X-Request-ID: {requestId}

{
  "contract_id": "CTR_20240101_12345",
  "applicant": {

    "education": "本科"
  },
```


## 5. 模板语法

### 5.1 变量引用

| 语法 | 描述 | 示例 |
|------|------|------|
| `${contractId}` | 合同ID | `"${contractId}"` → `"CTR_123"` |
| `${requestId}` | 请求ID | `"${requestId}"` → `"req_abc"` |
| `${input.xxx}` | 请求输入数据 | `${input.applicant.age}` |
| `${context.xxx}` | 上下文数据 | `${context.creditData.score}` |
| `${__timestamp__}` | 当前时间戳（毫秒） | `${__timestamp__}` → `1704067200000` |
| `${__datetime__}` | 当前时间（ISO8601） | `${__datetime__}` → `"2024-01-01T12:00:00Z"` |

### 5.2 模板示例

```yaml
inputMapping:
  # 简单变量
  contractId: "${contractId}"

  # 嵌套对象
  person:
    name: "${input.name}"
    age: "${context.applicant.age}"

  # 数组
  features:
    - "${context.feature1}"
    - "${context.feature2}"

  # 默认值
  optionalField: "${context.optionalField | default: 'N/A'}"

  # 类型转换
  ageStr: "${context.applicant.age | toString}"
  scoreNum: "${input.score | toFloat}"
```

## 6. 错误处理和降级

### 6.1 错误分类

| 错误类型 | 描述 | 处理方式 |
|----------|------|----------|
| 配置错误 | 模型配置无效 | 决策流终止，记录错误 |
| 网络错误 | 连接超时、DNS错误 | 重试 → 降级 → 失败 |
| 服务错误 | 5xx 响应 | 重试 → 降级 → 失败 |
| 客户端错误 | 4xx 响应 | 直接失败（请求参数问题） |
| 解析错误 | 响应格式错误 | 降级 → 失败 |

### 6.2 重试策略

```go
type RetryConfig struct {
    MaxAttempts  int           // 最大重试次数，默认3
    RetryDelay   time.Duration // 重试延迟，默认100ms
    MaxDelay     time.Duration // 最大延迟，默认2s
    BackoffMultiplier float64  // 退避倍数，默认2.0
}
```

重试流程：
```
首次请求
    │
    ├─ 成功 → 返回结果
    │
    └─ 失败 (可重试错误)
         │
         ├─ 已达最大重试次数?
         │    ├─ 是 → 降级或失败
         │    └─ 否 → 等待 RetryDelay * (BackoffMultiplier ^ attempt)
         │              │
         │              └─ 重试
```

### 6.3 降级策略

当模型服务不可用时，支持降级：

```yaml
fallback:
  enabled: true
  # 方式1: 固定默认值
  defaultOutput:
    defaultProbability: 0.5
    riskScore: 500
    riskLevel: "MEDIUM"
  # 方式2: 调用降级服务（可选）
  # fallbackService: "http://fallback-service/predict"
  # 方式3: 走简化规则（可选）
  # fallbackRules: ["R_FALLBACK_001"]
```

## 7. 监控和日志

### 7.1 日志记录

模型调用需要记录以下信息：

```go
type ModelInvokeLog struct {
    RequestID   string
    ModelID     string
    ModelName   string
    ContractID  string

    // 请求信息
    RequestURL     string
    RequestMethod  string
    RequestHeaders map[string]string
    RequestBody    string // 脱敏后

    // 响应信息
    ResponseStatus int

    ResponseTime   time.Duration

    // 结果
    Success    bool
    ErrorType  string
    ErrorMsg   string
    RetryCount int

    // 映射结果
    MappedOutput map[string]interface{}

```

### 7.2 指标统计

| 指标 | 说明 |
|------|------|
| `model_invoke_total` | 模型调用总次数 |

| `model_invoke_retry_total` | 重试次数 |

## 8. 决策流中的模型节点

### 8.1 模型节点配置

```json
{
  "id": "MODEL_1",
  "type": "MODEL",
  "modelId": "M001",

  // 输入字段映射（可选，覆盖模型配置中的 inputMapping）
  "inputMapping": {
    "contract_id": "${input.contractId}",
    "customField": "${context.someData}"
  },

  // 输出字段映射（可选，覆盖模型配置中的 outputMapping）
  "outputMapping": {
    "pd": "${modelOutput.defaultProbability}",
    "score": "${modelOutput.riskScore}"
  },

  // 输出到上下文的路径
  "outputToContext": "modelResults.M001",

  // 错误处理策略
  "errorPolicy": "CONTINUE",  // CONTINUE: 继续执行, FAIL: 终止决策
  "fallbackDecision": "MANUAL"  // 失败时的默认决策
}
```

### 8.2 上下文数据结构

执行模型节点后，上下文数据示例：

```json
{
  "input": {
    "contractId": "CTR_123",
    ...
  },
  "context": {
    "applicant": { ... },
    "creditData": { ... }

    "M001": {
      "success": true,
      "output": {
        "defaultProbability": 0.05,
        "riskScore": 720,
        "riskLevel": "LOW"
      },
      "executeTimeMs": 150
    }
  },
  "ruleResults": { ... }
}
```
